Hallazgo C-001 resuelto — Credenciales eliminadas del código fuente

Arquitectura de configuración implementada

Activo
📄
appsettings.json — Plantilla (repositorio)
Contiene estructura sin credenciales reales. Se sube al git. Incluye LogLevel y placeholder de connection string.
🔒
appsettings.Development.json — Real (gitignored)
Contiene la connection string real del servidor de desarrollo. Excluido del repositorio via .gitignore. Cada desarrollador crea su copia local.
🌍

🛠
ErpDbContextFactory — Migraciones design-time
Lee ERP_CONNECTION_STRING env var, con fallback a lectura del appsettings.Development.json relativo al proyecto. Sin strings hardcodeados.
⚠️
Acción pendiente: Rotar la contraseña del usuario sa una vez que todos los desarrolladores tengan el appsettings.Development.json local. La contraseña anterior quedó en el historial de git — considerar un git filter-branch o git-secrets para limpiar.

FASE 4 — PERFORMANCE EF ✅

N+1 Query Resuelto + SucursalId Fix

Hallazgos A-001 y A-002 resueltos

A-001 — N+1 en PedidoService.ListarAsync

Resuelto
⚡
Antes: subquery O(n) dentro del Select
Por cada pedido con cotización: _context.Cotizaciones.Where(...).FirstOrDefault() — generaba N+1 queries
✅
Ahora: .Include(p => p.Cotizacion)
Una sola query con LEFT JOIN. El folio se lee desde la propiedad de navegación: p.Cotizacion?.Folio

A-002 — SucursalId ?? 1 eliminado

Resuelto
✅
Validación explícita en GenerarDesdePedidoAsync
Si pedido.SucursalId es null, la operación retorna error descriptivo. Sin fallback implícito a Sucursal 1. El usuario recibe mensaje claro para resolver la situación.
